About the game

Game

This is a simple platformer game. I wanted to give a 2D vibe to a 3D game and add kind of mini-games, or a mashup of known games to dive into more hardware aspects. As this is not only playable on PC, but also on the Steamdeck, mobile devices, and in a web browser (try it out on Itch.io). I made this project by myself in 4 weeks. During this time I mainly focused on implementing the SOLID principles, I had a lot of fun in better understanding how to make my game more flexible!

    New Input System: Jump

    First time using Unity's new input system. Here is a snippet of the jump logic In the OnEnable I subscribe this action to the input. I wanted the player to be able to double jump, so I have an int that keeps track of the amount of jumps. OnJump I add force to the player in the upwards direction. Then I let my own gravity weigh the player down. Once it reaches the ground, the jumps are reset.

    Inventory

    Here is a simple inventory system, it has only one slot, because I wanted the player to only hold one item at a time (it's just a small cat). So there is an Add and Remove function, which adds only an Scriptable Object 'item'. Upon adding or removing an item I update the inventory slot's icon.

    Interface

    I used an Interface for the interactables. The dynamite is one of the interactables In this part I add the Dynamite to the inventory. This is an object that gets instantiated after the chest has been opened; it will float above the chest.

    OnInteract

    Using the new input system I check for objects that have an interactable Interface when the right key is pressed.
    However, I also constantly check for interactables to give the player feedforward to indicate an object can be interacted with. I do this by enabling a canvas and an outline shader.
